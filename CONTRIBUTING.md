# Contributing

Wayfindr is early-stage. The contribution model will evolve once the first working prototype exists.

## Public-First Product Work

Contributions should be suitable for a public open source repository. Do not include:

- customer or prospect names,
- private business strategy,
- pricing experiments,
- credentials or secrets,
- private infrastructure details,
- commercially sensitive planning notes.

See [docs/governance/public-information-policy.md](docs/governance/public-information-policy.md).

Issues and pull requests should describe public product, code, documentation, or self-hosting concerns. Keep private go-to-market context and customer-specific details out of the repository.

## License Boundaries

The core application is licensed as `AGPL-3.0-or-later`. SDKs and integration packages may use separate permissive licenses. Check the nearest `LICENSE` file before reusing code outside this repository.

Before accepting substantial external contributions, this project may adopt either a Contributor License Agreement or Developer Certificate of Origin process. Until then, maintainers should limit accepted external changes to small corrections that do not create unclear ownership.
